3. Create `internal/config/config.go` with these exact types:

   ```go
   package config

   type Config struct {
       Timezone string `yaml:"timezone"`
       Schedule string `yaml:"schedule"`
       Rules    []Rule `yaml:"rules"`
   }

   type Rule struct {
       Name       string      `yaml:"name"`
       Conditions *Conditions `yaml:"conditions"`
       Unless     *Exceptions `yaml:"unless"`
       Action     string      `yaml:"action"`
   }

   type Conditions struct {
       OlderThan  *int    `yaml:"olderThan"`
       Source     *string `yaml:"source"`
       Archived   *bool   `yaml:"archived"`
       Favourited *bool   `yaml:"favourited"`
       HasTag     *string `yaml:"hasTag"`
       LacksTag   *string `yaml:"lacksTag"`
   }

   type Exceptions struct {
       Favourited *bool   `yaml:"favourited"`
       HasTag     *string `yaml:"hasTag"`
       HasNote    *bool   `yaml:"hasNote"`
       Archived   *bool   `yaml:"archived"`
   }
   ```

   All optional fields use pointer types (`*int`, `*string`, `*bool`) so nil means "not specified" and zero-value means "explicitly set".

4. Implement `Load(path string) (*Config, error)`:
   - `os.Open(path)` -- on error wrap: `fmt.Errorf("opening config: %w", err)`
   - `defer f.Close()`
   - `yaml.NewDecoder(f)` with `decoder.KnownFields(true)` for CONF-02 unknown field rejection
   - `decoder.Decode(&cfg)` -- on error wrap: `fmt.Errorf("parsing config: %w", err)`
   - On decode error do NOT use the partially populated struct (Pitfall 3 from research)
   - Return `&cfg, nil` on success (validation comes in Plan 02)

5. Implement `ResolvePath(flagValue string) string`:
   - If `flagValue != ""` return flagValue
   - If `os.Getenv("KARACLEAN_CONFIG") != ""` return that
   - Otherwise return `"/config/karaclean.yaml"`

IMPORTANT: Do NOT use custom `UnmarshalYAML` methods -- they break `KnownFields` (Pitfall 2 from research). Use plain struct tags only.
