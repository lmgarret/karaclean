<objective>
Implement error notification on invalid config: when config validation fails at startup, send a notification to the configured default channel before exiting — if the user has opted in via `notifyOnError: true` in the notifications section.

Purpose: Users running karaclean as an unattended Docker sidecar need to know when their config is broken, without checking container logs. This surfaces startup failures via their existing notification channel (ntfy, Slack, Telegram).

Output: Working two-pass config loader that dispatches error notifications on validation failure, with lenient fallback for YAML syntax errors, full test coverage, and updated docs.
</objective>

    ```go
    // SendConfigError sends a config validation error notification via the default channel.
    // It is best-effort: if sending fails, the error is logged but not returned.
    // notifier may be any engine.Notifier implementation (production or mock).
    // This function accepts an interface matching engine.Notifier to avoid import cycles:
    //   type ConfigErrorNotifier interface { Send(url, message, title string) error }
    func SendConfigError(n *Notifications, notifier ConfigErrorNotifier, configErr error) {
    ```
    - Define `ConfigErrorNotifier` interface in config package (mirrors `engine.Notifier`) to avoid importing engine from config (would create import cycle: config -> engine -> config).
    - If `n` is nil, return immediately.
    - If `n.NotifyOnError` is nil or `*n.NotifyOnError == false`, return immediately.
    - If `n.Default == ""`, return immediately (no channel to send to, per D-04).
    - Look up `n.Channels[n.Default]` to get URL. If channel not found, return.
    - Title: `[ERROR] [karaclean] config validation failed` (per D-05).
    - Body: `configErr.Error()` (the validation error messages).
    - Call `notifier.Send(url, body, title)`. If Send returns error, `log.Printf("WARNING: failed to send config error notification: %v", err)`.

    **3. Refactor Load for two-pass approach (D-02):**
    Change `Load` signature to accept an optional notifier:
    ```go
    func Load(path string, notifier ...ConfigErrorNotifier) (*Config, error)
    ```
    Variadic notifier parameter maintains backward compatibility — existing callers pass no notifier.

    Pass 1: YAML decode into `cfg`. If decode succeeds, proceed to validation.
    Pass 2: `cfg.Validate()`. If validation fails AND `cfg.Notifications` is non-nil:
    ```go
    if errs := cfg.Validate(); len(errs) > 0 {
        validationErr := &ValidationErrors{Errors: errs}
        if len(notifier) > 0 && notifier[0] != nil {
            SendConfigError(cfg.Notifications, notifier[0], validationErr)
        }
        return nil, validationErr
    }
    ```
