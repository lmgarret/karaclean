# Milestone v1.1: Notifications

**Status:** ✅ SHIPPED 2026-03-20
**Phases:** 01
**Total Plans:** 3

## Overview

Added per-rule notification dispatch via Shoutrrr to configurable channels (Slack, ntfy, Telegram, etc.) with global default channel and per-rule override. Notifications fire after each rule's bookmark evaluation, delivering action summaries with deleted/archived counts. Best-effort delivery — failures are logged, not fatal.

## Milestone Summary

**Key Decisions:**

- `Notifications` is `*Notifications` (nil = opt-in disabled, no validation errors)
- Shoutrrr URL validated at config load time via `CreateSender` (fail-fast)
- `Notifier` interface uses `Send(url, message, title)` for testability
- `main.go` creates notifier only when `cfg.Notifications` is non-nil (nil = no Shoutrrr overhead)
- `Run()` signature extended with trailing params `(notifications, notifier)` for backward compat
- Notification body uses `Summary:` prefix (not title duplication) — improved in quick task 260320-ls1

**Issues Resolved:**

- HTTP 204 treated as error on bookmark delete (fixed in quick task 260320-lfo)
- Notification message body duplicated the title (fixed in quick task 260320-ls1)

**Issues Deferred:**
