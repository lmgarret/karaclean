2. A `NewClient` constructor that takes baseURL and apiKey:
   - Create bearer auth via `securityprovider.NewSecurityProviderBearerToken(apiKey)`
   - Append `/api/v1` to baseURL when constructing the generated client (Pitfall 1 from research)
   - Return `*Client` and error
   ```go
   func NewClient(baseURL, apiKey string) (*Client, error) {
       bearerAuth, err := securityprovider.NewSecurityProviderBearerToken(apiKey)
       if err != nil {
           return nil, fmt.Errorf("creating auth provider: %w", err)
       }
       inner, err := NewClientWithResponses(
           baseURL+"/api/v1",
           WithRequestEditorFn(bearerAuth.Intercept),
       )
       if err != nil {
           return nil, fmt.Errorf("creating API client: %w", err)
       }
       return &Client{inner: inner}, nil
   }
   ```

3. `CheckAuth` method:
   - Call `c.inner.GetCurrentUserWithResponse(ctx)` (the generated method for GET /users/me -- verify exact method name from client.gen.go)
   - On network error: return `fmt.Errorf("auth check: %w", err)`
   - On 401 status: return `fmt.Errorf("authentication failed: invalid API token (check KARAKEEP_API_KEY)")`
   - On non-200 status: return `fmt.Errorf("auth check: unexpected status %d", resp.StatusCode())`
   - On 200: return nil

4. `ListBookmarks` method with cursor-based pagination:

     - Parse createdAt to time.Time if the generated type is string
     - Map archived, favourited, source, note fields
     - Handle note field: if nil/empty in generated type, set to empty string

5. Add compile-time interface check:
