### 2. What is Karaclean? (2-3 paragraphs)
- Explain the problem: Karakeep accumulates bookmarks, manual cleanup is tedious
- Explain the solution: define YAML rules with conditions, exceptions, actions; Karaclean runs on a cron schedule as a sidecar
- Key safety features: dry-run mode, exception clauses to protect important bookmarks, strict config validation rejects typos

### 3. Quick Start
Step-by-step for docker-compose users:
1. Get a Karakeep API key (Settings > API Keys in Karakeep UI)
2. Create `karaclean.yaml` -- show a minimal 10-line example:
```yaml
schedule: "0 3 * * *"
rules:
  - name: archive-old-rss
    conditions:
      olderThan: "30d"
      source: rss
    unless:
      favourited: true
    action: archive
```
3. Create `docker-compose.yml` snippet showing karaclean service alongside karakeep:
```yaml
services:
  karaclean:
    image: ghcr.io/lm/karaclean:latest  # placeholder registry path
    environment:
      - KARAKEEP_URL=http://karakeep:3000
      - KARAKEEP_API_KEY=your-api-key-here
    volumes:
      - ./karaclean.yaml:/config/karaclean.yaml:ro
    depends_on:
      - karakeep
    restart: unless-stopped
```
4. `docker compose up -d`
5. Check logs: `docker compose logs karaclean`

### 4. Configuration Reference
Full table-based documentation of every config field:

**Top-level fields:**
| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `schedule` | string | Yes | - | Cron expression (5-field: minute hour day month weekday) |
| `timezone` | string | No | UTC (with warning) | IANA timezone name (e.g., `America/New_York`) |
| `dryRun` | bool | No | `false` | Log actions without executing mutations |
| `rules` | list | Yes | - | List of cleanup rules |

**Rule fields:**
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | Human-readable rule identifier (used in logs) |
| `conditions` | object | Yes | Matching criteria (all must match -- AND semantics) |
| `unless` | object | No | Exception criteria (any match skips -- OR semantics) |
| `action` | string | Yes | `archive` or `delete` |

**Conditions (all optional, AND semantics):**
| Field | Type | Example | Description |
|-------|------|---------|-------------|
| `olderThan` | string | `"30d"` | Match bookmarks created more than N ago. Formats: `Nh` (hours), `Nd` (days), `Nw` (weeks), `Nmo` (months=30d), `Ny` (years=365d). Strictly-greater-than (exact boundary does not match). |
| `source` | string | `"rss"` | Match bookmarks from this source. Values: `rss`, `web`, `api`, `mobile`, `extension`, `cli`, `import` |
| `archived` | bool | `true` | `true` = only archived, `false` = only non-archived |
| `favourited` | bool | `true` | `true` = only favourited, `false` = only non-favourited |
| `hasTag` | string | `"read-later"` | Match bookmarks that have this exact tag (case-sensitive) |
| `lacksTag` | string | `"keep"` | Match bookmarks that do NOT have this tag (case-sensitive) |

**Exceptions (all optional, OR semantics):**
| Field | Type | Example | Description |
|-------|------|---------|-------------|
| `favourited` | bool | `true` | Skip if bookmark matches this star status |
| `hasTag` | string | `"important"` | Skip if bookmark has this tag |
| `hasNote` | bool | `true` | Skip if bookmark has a personal note (whitespace-only counts as empty) |
| `archived` | bool | `true` | Skip if bookmark matches this archive status |

### 5. Rule Examples
Show 4-5 practical rule examples with explanations:
1. "Archive old RSS after 30 days, protect favourites" (basic)
2. "Delete ancient archived bookmarks after 90 days, protect tagged and noted" (multi-exception)
3. "Archive stale untagged bookmarks after 60 days" (lacksTag + favourited condition)
4. "Delete all web bookmarks older than 1 year" (simple aggressive cleanup)
5. "Archive mobile bookmarks older than 2 weeks unless they have notes" (short retention)

### 6. Rule Evaluation
Explain how rules work:
- First-match-wins: rules are evaluated in order, first matching rule applies
- Collect-then-act: all bookmarks are fetched first, then rules are evaluated (prevents pagination race conditions)
- If no rule matches a bookmark, it is left untouched
- Exception check happens after condition match but before action execution

### 7. Dry-Run Mode
- What it does: logs all intended actions without executing mutations
- Three ways to enable with precedence order:
  1. `--dry-run` CLI flag (highest)
  2. `KARACLEAN_DRY_RUN=true` environment variable
  3. `dryRun: true` in YAML config (lowest)
- Recommend running dry-run first when setting up new rules

### 8. CLI Reference
| Flag | Default | Description |
|------|---------|-------------|
| `--config PATH` | (see resolution) | Path to YAML config file |
| `--dry-run` | `false` | Enable dry-run mode |

**Config path resolution** (first match wins):
1. `--config PATH` flag
2. `KARACLEAN_CONFIG` environment variable
3. `/config/karaclean.yaml` (default, matches Docker volume mount)

### 9. Environment Variables
| Variable | Required | Description |
|----------|----------|-------------|
| `KARAKEEP_URL` | Yes | Karakeep instance URL (e.g., `http://karakeep:3000`) |
| `KARAKEEP_API_KEY` | Yes | API key from Karakeep Settings > API Keys |
| `KARACLEAN_CONFIG` | No | Config file path (overrides default `/config/karaclean.yaml`) |
| `KARACLEAN_DRY_RUN` | No | Set to `true` or `1` to enable dry-run mode |

### 10. Docker
- **Image**: Built from scratch (minimal, ~10MB), static Go binary
- **Volumes**: Mount config at `/config/karaclean.yaml` (read-only recommended)
- **Networking**: Needs access to Karakeep API URL
- **Signals**: Responds to SIGTERM and SIGINT for graceful shutdown
- **Timezone**: Embedded timezone database (no need to mount `/usr/share/zoneinfo`)

Show the Dockerfile build command:

```

### 12. Observability
- Startup: logs auth check, dry-run status, timezone, next scheduled run
- Each run logs: `run complete: archived=N deleted=M no_match=K excepted=J errors=E`
- Runs on startup (immediate first run) then on cron schedule
- Overlapping runs are automatically skipped (SkipIfStillRunning)

```
Note: requires Go 1.26+

- Missing required fields (name, conditions, action) produce clear error messages
- Invalid enum values (source, action) are caught before any rules execute
- All validation errors are collected and reported together (not one-at-a-time)
