2. Create `.github/workflows/ci.yml` with the following exact content:

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

concurrency:
  group: ${{ github.workflow }}-${{ github.head_ref || github.ref }}
  cancel-in-progress: true

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6
      - uses: actions/setup-go@v6
        with:
          go-version-file: go.mod
      - run: go test -race ./...

  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6
      - uses: actions/setup-go@v6
        with:
          go-version-file: go.mod
      - uses: golangci/golangci-lint-action@v9
        with:
          version: v2.11

  docker:
    needs: [test, lint]
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write
    steps:
      - uses: actions/checkout@v6
      - uses: docker/setup-buildx-action@v3
      - uses: docker/login-action@v4
        if: github.event_name == 'push' && github.ref == 'refs/heads/main'
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}
      - uses: docker/metadata-action@v6
        id: meta
        with:
          images: ghcr.io/${{ github.repository }}
          tags: |
            type=raw,value=latest,enable=${{ github.ref == 'refs/heads/main' }}
            type=sha,prefix=
      - uses: docker/build-push-action@v6
        with:
          context: .
          push: ${{ github.event_name == 'push' && github.ref == 'refs/heads/main' }}
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
```

Key design decisions per CONTEXT.md (all locked):
- Triggers: push to main + PR targeting main (no push-to-all-branches)
- Concurrency: `github.head_ref || github.ref` gives unique group per branch/PR; cancel-in-progress saves CI minutes
- `go-version-file: go.mod` auto-reads `go 1.26.1` -- no hardcoded version
- `actions/setup-go@v6` has built-in caching for `~/go/pkg/mod` and `~/.cache/go-build` -- no separate `actions/cache` step
- `go test -race ./...` runs all tests with race detector (CGO enabled by default on ubuntu-latest)
- `golangci-lint-action@v9` handles binary download and caching; `version: v2.11` pins minor version
- Docker job: `needs: [test, lint]` -- only runs after both pass (fail fast)
- Docker login gated on `github.event_name == 'push' && github.ref == 'refs/heads/main'` -- prevents fork PR failures
- `push:` parameter uses same condition -- builds on PRs (verifies compilation) but only pushes on main
- `permissions: { contents: read, packages: write }` required for GHCR push (pitfall 2 from research)
- Tags: `type=raw,value=latest` for latest tag (only on main), `type=sha,prefix=` for git SHA tag
- Image name: `ghcr.io/${{ github.repository }}` resolves to `ghcr.io/lm/karaclean` (all lowercase)
